## 言語選択戦略

| レイヤー | 言語 | 理由 |
|---------|------|------|
| Frontend | TypeScript | 高速開発、AIで安定 |
| Service | TypeScript | 互換性、生産性 |
| Core | Rust | 性能要件高い場合 |
| Scripts | Python | 運用・分析用 |

## デプロイメント

- **ローカル**: Docker Compose
- **本番**: Railway
- **CI/CD**: GitHub Actions
